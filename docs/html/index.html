ATREX

Introduction

Advanced Tools for eXtreme Xtallography (ATREX) is an open source software package originally developed to process single crystal x-ray diffraction data.

Author :

Przemyslaw Dera, Hawaii Institute of Geophysics and Planetology

Installation

The software is written and run in Python. Requirements include Python 2.7 and several other mathematical, statistical and graphical python packages. These packages include the following

Python 2.7
matplotlib
numpy
scipy
PyQt4

Installing these packages can be a long and frustrating task. An easier way for a user to fulfill these requirements is to download and install the open source Anaconda python platform. Options exist for Linux or Windows and 32 or 64 bit distributions. Software available at https://www.continuum.io/downloads. The Atrex software is available via github, and can be installed by downloading a .zip file or via git clone. In a browser, a user can navigate to https://github.com/comptech/atrex and then obtain the source code and demo data by selecting the green “Clone or download” tab. Choose a suitable directory location to clone or unpack the zip file, which will then be the the root directory of the installation.

Program Structure and Files

Atrex is a graphical user interface (GUI) based package, with processing guided by user's interactions with various controls of the interface. The program uses the PyQt4 toolkit to construct this interface. The construction of the interface and the preliminary handling of these interactions or events is handled by the Atrex class, which is defined in the Atrex.py file. Other major classes and files and their functionality include the following

Atrex : Atrex.py
Top level class of the Atrex program, constructs the main window, and handles all events generated from interacting with controls on that window. The interface is defined as Atrex.ui with controls being Atrex.ui.*.
Project : Project.py
This is a bookkeeping class, which determines the min and max of an image series, the image base and geometries related to each file. Used primarily by Atrex when a user wishes to navigate through a defined image series.
myImage : myImage.py
Class to handle Atrex images, prepare images for display and coversion to a numpy array. Files can be either Tiff or HDF5. Important class methods include readTiff and readHDF5 for file reading and readText for reading the image's associated text file to extract the omega0, omegaR, chi, detector and exposure time values. Also, a number of methods are instrumental in the peak search and peak fitting routines, such as search_for_peaks, calculate_local_background, smoothGauss, fitPeaks and fitAllPeaks. In addition, this class is responsible for integrating and creating the cake image.
myDetector : myDetector.py
Class to handle the detector geometry and characteristics. Numerous methods for getting detector parameters including getdist, getwavelength, gettiltom, gettiltch, getbeamXY, and getpsizeXY. Also methods to read from and write to an ASCII file. Numerous functions to calculate parameters based upon the other detector parameters. This class is used by most other classes as detector geometry is vital information to conduct a variety of analyses.
myPeak : myPeakTable.py
myPeakTable : myPeakTable.py
The myPeakTable class is used to define and manipulate the peaks found and fitted for an image and / or series of images. Methods to clear, select, delete, and move peaks from the inactive to active list are defined here. Note that each peak is a "myPeak" object, which is also defined in the myPeakTable.py file.
peakFit : peakFit.py
The peakFit class is used to calculate the fit of a two dimension gaussian to a selected peak. The use of this class has given way to the fitPeak method in myImage.py.
myImDisplay : myImDisplay.py
myZmDisplay : myZmDisplay.py
myZmPeakDisplay : myZmPeakDisplay.py
These classes are used for image display in windows of the user interface. They are used to display the images with graphics overlays for peak display, selection, and manipulation.
crystallography : crystallography.py
crystallography module with basic functions for x-ray crystallography.

User Interface Files

User interface files, those which are used to construct the interface and its controls are created and editted uses the QtDesigner program, which features a visual designer to easily manipulate the container window and its constituent widgets. Therefore, the .ui files which are created by the program are not directly editted. Instead, the developer opens the .ui file with the QtDesigner program, visually edits the interface, then saves the work, writing a new .ui file. Besides the main window, other dialogs are created to get additional user input beyond the main window and at advanced stages of analysis. The ui files in the Atrex program include...
uiMainWin.ui
This is the main window of the Atrex program. This ui and all interactions with widgets on this interface are controlled by the Atrex.py file.
genSettingsDlg.ui
This creates a "settings" dialog, which is opened when an image is opened and a settings file does not exist in that directory. Created and controlled by th myGenSettings.py file and class.
peakEditDlg.ui
This creates the "Peak Edit" dialog, which is opened when a peak in the peak list table is double clicked. Allows for properties of the peak to be manipulated. Loaded and controlled by peakEditDlg.py.
peakDisplayAdjust.ui
This creates the "Peak Display Adjust" dialog. Allows for the scaling of the peak display images to be adjusted to better highlight the intensity of the peak and its surrounding pixels. Loaded and controlled by peakDisplayAdjust.py.
cellPathDlg.ui
Gets the absolute file path to the directory containing the cell_now executable.
simulateDlg.ui
Gets additional user controlled parameters for the predict, simulate and indexing. Loaded and controlled by simulateDlg.py.
